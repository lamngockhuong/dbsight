DBSight thu thập một Snapshot số liệu cấp cao mỗi lần Worker chạy cho mỗi cơ sở dữ liệu. Các số liệu này cung cấp dữ liệu cho các thẻ tóm tắt và biểu đồ sparkline ở đầu Bảng điều khiển trực tiếp.

## Thống kê Cơ sở dữ liệu PostgreSQL

Nguồn: `internal/adapter/postgres_stats.go` — `GetDatabaseStats`

```sql
SELECT
    current_database(),
    pg_database_size(current_database()),
    (SELECT count(*) FROM pg_stat_activity WHERE state = 'active'),
    current_setting('max_connections')::int,
    ROUND(
        sum(blks_hit)::numeric /
        NULLIF(sum(blks_hit) + sum(blks_read), 0) * 100, 2
    )
FROM pg_stat_database WHERE datname = current_database()
```

Truy vấn chọn từ `pg_stat_database` được lọc theo cơ sở dữ liệu hiện tại, kết hợp với một subquery tương quan để lấy số kết nối đang hoạt động và một lệnh gọi `current_setting` để lấy giới hạn kết nối đã cấu hình.

### Tài liệu Tham chiếu Số liệu

#### Tên Cơ sở dữ liệu

```sql
current_database()
```

Trả về tên của cơ sở dữ liệu mà kết nối đang được xác thực. Được dùng làm nhãn hiển thị trong tiêu đề bảng điều khiển.

#### Kích thước Cơ sở dữ liệu

```sql
pg_database_size(current_database())
```

Trả về tổng kích thước trên đĩa của cơ sở dữ liệu tính bằng byte, bao gồm tất cả các bảng, chỉ mục và dữ liệu TOAST. DBSight định dạng giá trị này thành các đơn vị dễ đọc (KB / MB / GB) để hiển thị.

#### Kết nối Đang hoạt động

```sql
(SELECT count(*) FROM pg_stat_activity WHERE state = 'active')
```

Đếm các kết nối hiện đang thực thi một truy vấn (`state = 'active'`). Điều này loại trừ các kết nối idle, kết nối idle-in-transaction và các background worker.

| `pg_stat_activity.state` | Ý nghĩa                                                  |
| ------------------------ | -------------------------------------------------------- |
| `active`                 | Đang thực thi một truy vấn ngay lúc này — được đếm ở đây |
| `idle`                   | Đã kết nối nhưng đang chờ lệnh                           |
| `idle in transaction`    | Trong khối BEGIN, hiện không thực thi                    |
| `fastpath function call` | Đang thực thi một hàm fast-path                          |

#### Số kết nối Tối đa

```sql
current_setting('max_connections')::int
```

Giới hạn kết nối toàn server từ `postgresql.conf`. DBSight hiển thị số kết nối đang hoạt động cùng với giá trị này để bạn có thể thấy dư địa còn lại ngay lập tức.

#### Tỷ lệ Cache Hit

```sql
ROUND(
    sum(blks_hit)::numeric /
    NULLIF(sum(blks_hit) + sum(blks_read), 0) * 100, 2
)
```

Tỷ lệ phần trăm các lần đọc khối được phục vụ từ bộ nhớ đệm dùng chung của PostgreSQL thay vì từ đĩa. `NULLIF` bảo vệ khỏi chia cho số không trên cơ sở dữ liệu mới chưa có I/O.

| Khoảng    | Trạng thái    | Diễn giải                                                                                   |
| --------- | ------------- | ------------------------------------------------------------------------------------------- |
| >= 99%    | Tốt           | Hầu hết các lần đọc từ bộ nhớ. Dự kiến cho cơ sở dữ liệu được cấu hình tốt và đã khởi động. |
| 95% – 99% | Cảnh báo      | Một số lần đọc đĩa đang xảy ra. Có thể chấp nhận được tùy thuộc vào kích thước dataset.     |
| < 95%     | Nghiêm trọng  | I/O đĩa đáng kể. Cân nhắc tăng `shared_buffers` hoặc bổ sung RAM.                           |
| `NULL`    | Không áp dụng | Chưa ghi nhận I/O khối — cơ sở dữ liệu vừa khởi động hoặc thống kê đã được reset.           |

Tỷ lệ cache hit thấp không phải lúc nào cũng có nghĩa là cấu hình sai. Các cơ sở dữ liệu lớn hơn RAM khả dụng sẽ luôn có một số lần đọc đĩa; điều quan trọng là liệu tỷ lệ có đang giảm theo thời gian hay không.

## Thống kê Cơ sở dữ liệu MySQL

MySQL không cung cấp một view hệ thống duy nhất cho tất cả các số liệu cấp cơ sở dữ liệu. DBSight phát nhiều truy vấn tới `SHOW GLOBAL STATUS`, `SHOW VARIABLES` và `information_schema.TABLES`.

Nguồn: `internal/adapter/mysql_stats.go` — `GetDatabaseStats`

### Tên Cơ sở dữ liệu

```sql
SELECT DATABASE()
```

Trả về tên của cơ sở dữ liệu đang được chọn hiện tại.

### Kích thước Cơ sở dữ liệu

```sql
SELECT IFNULL(SUM(data_length + index_length), 0)
FROM information_schema.TABLES
WHERE table_schema = DATABASE()
```

Tổng hợp `data_length` (dữ liệu hàng) và `index_length` (dữ liệu chỉ mục) cho tất cả các bảng trong cơ sở dữ liệu hiện tại. Đây là ước tính dựa trên thống kê InnoDB; chạy `ANALYZE TABLE` trên các bảng quan trọng để có giá trị chính xác hơn.

### Kết nối Đang hoạt động

```sql
SHOW GLOBAL STATUS LIKE 'Threads_connected'
```

Trả về số lượng kết nối client đang mở hiện tại. Khác với `pg_stat_activity` của PostgreSQL, lệnh này đếm tất cả các thread đã kết nối bất kể trạng thái hiện tại của chúng (đang hoạt động, idle, đang chờ).

### Số kết nối Tối đa

```sql
SHOW VARIABLES LIKE 'max_connections'
```

Giới hạn kết nối toàn server từ `my.cnf` hoặc được đặt tại runtime bằng `SET GLOBAL max_connections`.

### Tỷ lệ Cache Hit InnoDB Buffer Pool

DBSight tính tỷ lệ cache hit từ hai biến trạng thái:

```sql
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_reads'
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_read_requests'
```

Tỷ lệ được tính như sau:

```
cache_hit_ratio = (1 - Innodb_buffer_pool_reads / Innodb_buffer_pool_read_requests) * 100
```

| Biến                               | Mô tả                                     |
| ---------------------------------- | ----------------------------------------- |
| `Innodb_buffer_pool_reads`         | Lần đọc đĩa vật lý (cache miss)           |
| `Innodb_buffer_pool_read_requests` | Tổng yêu cầu đọc logic (cache hit + miss) |

Đây là các bộ đếm tích lũy kể từ khi server khởi động. Tỷ lệ cao (> 99%) cho thấy hầu hết các lần đọc được phục vụ từ InnoDB buffer pool.

| Khoảng    | Trạng thái    | Diễn giải                                                                   |
| --------- | ------------- | --------------------------------------------------------------------------- |
| >= 99%    | Tốt           | Hầu hết các lần đọc từ bộ nhớ buffer pool.                                  |
| 95% – 99% | Cảnh báo      | Một số lần đọc đĩa. Có thể chấp nhận được tùy thuộc vào kích thước dataset. |
| < 95%     | Nghiêm trọng  | I/O đĩa đáng kể. Cân nhắc tăng `innodb_buffer_pool_size`.                   |
| `0`       | Không áp dụng | Chưa có yêu cầu đọc — server vừa khởi động hoặc chưa truy cập bảng InnoDB.  |

## Thống kê Cơ sở dữ liệu MariaDB

MariaDB sử dụng cùng cách tiếp cận truy vấn như MySQL. Cả hai engine đều chia sẻ giao diện `SHOW GLOBAL STATUS`, `SHOW VARIABLES` và `information_schema.TABLES`.

Nguồn: `internal/adapter/mariadb_stats.go` — `GetDatabaseStats`

### Tên Cơ sở dữ liệu

```sql
SELECT DATABASE()
```

### Kích thước Cơ sở dữ liệu

```sql
SELECT IFNULL(SUM(data_length + index_length), 0)
FROM information_schema.TABLES
WHERE table_schema = DATABASE()
```

### Kết nối Đang hoạt động

```sql
SHOW GLOBAL STATUS LIKE 'Threads_connected'
```

### Số kết nối Tối đa

```sql
SHOW VARIABLES LIKE 'max_connections'
```

### Tỷ lệ Cache Hit InnoDB Buffer Pool

```sql
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_reads'
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_read_requests'
```

Tính giống hệt MySQL:

```
cache_hit_ratio = (1 - Innodb_buffer_pool_reads / Innodb_buffer_pool_read_requests) * 100
```

### So sánh MariaDB và MySQL

Các truy vấn thống kê cơ sở dữ liệu về mặt chức năng giống nhau giữa MySQL và MariaDB. Cả hai engine đều cung cấp cùng các biến `SHOW GLOBAL STATUS` và schema `information_schema.TABLES`. Không có sự khác biệt hành vi nào cho tập số liệu này.

Một lưu ý vận hành: MariaDB có thể báo `Threads_connected` hơi khác MySQL khi thread pooling được bật (`thread_handling = pool-of-threads`). Trong cấu hình đó, `Threads_connected` phản ánh các kết nối logic thay vì các OS thread.

## Cách DBSight Hiển thị các Số liệu Này

Bốn giá trị ánh xạ trực tiếp đến các thẻ tóm tắt trên Bảng điều khiển trực tiếp:

- **Cơ sở dữ liệu** — nhãn tên trong tiêu đề trang
- **Kích thước** — thẻ byte đã định dạng (ví dụ: `4.2 GB`)
- **Kết nối** — `{active} / {max_connections}` với thanh sử dụng
- **Cache Hit** — tỷ lệ phần trăm với mã màu (xanh lá / hổ phách / đỏ theo các khoảng trên)

Cả bốn giá trị đều làm mới theo mỗi tick SSE cùng với luồng truy vấn chậm, nên bảng điều khiển phản ánh trạng thái trực tiếp của cơ sở dữ liệu mà không cần tải lại trang thủ công.
