DBSight uses an adapter pattern to isolate all database-specific logic behind a single interface. Adding support for a new database type (MySQL, SQLite, CockroachDB, etc.) means creating one new file and registering it in the factory — no changes to the API, worker, or store packages.

## The DBAnalyzer Interface

Every adapter must implement all methods in `internal/adapter/adapter.go`:

```go
type DBAnalyzer interface {
    Connect(ctx context.Context, dsn string) error
    Close() error
    GetSlowQueries(ctx context.Context, opts QueryOpts) ([]models.SlowQuery, error)
    GetExplainPlan(ctx context.Context, query string, opts QueryOpts) (*models.ExplainPlan, error)
    GetIndexStats(ctx context.Context) ([]models.IndexStat, error)
    GetTableStats(ctx context.Context) ([]models.TableStat, error)
    GetDuplicateIndexes(ctx context.Context) ([]models.DuplicateIndex, error)
    GetDatabaseStats(ctx context.Context) (*models.DatabaseStats, error)
}
```

### Method Descriptions

| Method                | Description                                                |
| --------------------- | ---------------------------------------------------------- |
| `Connect`             | Open a connection pool using the decrypted DSN             |
| `Close`               | Release all connections gracefully                         |
| `GetSlowQueries`      | Return top slow queries, analogous to `pg_stat_statements` |
| `GetExplainPlan`      | Run `EXPLAIN` on a query; use a read-only transaction      |
| `GetIndexStats`       | Per-index usage stats (scans, rows read, size)             |
| `GetTableStats`       | Per-table sequential scan and row counts                   |
| `GetDuplicateIndexes` | Find indexes with identical definitions                    |
| `GetDatabaseStats`    | DB name, size, connection counts, cache hit ratio          |

`QueryOpts` carries collection settings:

```go
type QueryOpts struct {
    Limit       int
    MinMeanMs   float64
    AnalyzeMode bool  // include ANALYZE in EXPLAIN when true
}
```

## Real Examples: MySQL and MariaDB Adapters

DBSight includes production implementations for MySQL and MariaDB. Here's how they're structured:

### MySQL Adapter Structure

- `internal/adapter/mysql.go` — Connection and core methods
- `internal/adapter/mysql_slow_queries.go` — Queries `performance_schema.events_statements_summary_by_digest`
- `internal/adapter/mysql_explain.go` — Handles `EXPLAIN FORMAT=JSON` and `EXPLAIN ANALYZE TREE FORMAT=JSON`
- `internal/adapter/mysql_indexes.go` — Queries `information_schema.statistics` with metrics from `performance_schema`
- `internal/adapter/mysql_stats.go` — Database-level stats from `SHOW GLOBAL STATUS`
- `internal/adapter/mysql_explain_test.go` — Tests for EXPLAIN output parsing

### MariaDB Adapter Structure

- `internal/adapter/mariadb.go` — Connection and core methods
- `internal/adapter/mariadb_slow_queries.go` — Queries `performance_schema` (same as MySQL, but handles missing fields)
- `internal/adapter/mariadb_explain.go` — Handles `EXPLAIN FORMAT=JSON` and `ANALYZE FORMAT=JSON`
- `internal/adapter/mariadb_indexes.go` — JSON-based index queries with `JSON_ARRAYAGG`
- `internal/adapter/mariadb_stats.go` — Database-level stats
- `internal/adapter/mariadb_explain_test.go` — Tests for MariaDB-specific EXPLAIN formats

### Shared Utilities

Both adapters use helpers from `internal/adapter/mysqlcompat/`:

- `helpers.go` — DSN builder, EXPLAIN JSON parsing, performance_schema conversions (picoseconds to ms)

## Step-by-Step: Adding a New Adapter (e.g., SQLite)

### 1. Add the driver dependency

```bash
go get github.com/mattn/go-sqlite3
```

### 2. Create the adapter files

Create `internal/adapter/sqlite.go`:

```go
package adapter

import (
    "context"
    "database/sql"
    "fmt"

    _ "github.com/mattn/go-sqlite3"
    "github.com/lamngockhuong/dbsight/internal/models"
)

type SQLiteAdapter struct {
    db *sql.DB
}

func NewSQLiteAdapter() *SQLiteAdapter {
    return &SQLiteAdapter{}
}

func (a *SQLiteAdapter) Connect(ctx context.Context, dsn string) error {
    db, err := sql.Open("sqlite3", dsn)
    if err != nil {
        return fmt.Errorf("sqlite open: %w", err)
    }
    if err := db.PingContext(ctx); err != nil {
        return fmt.Errorf("sqlite ping: %w", err)
    }
    a.db = db
    return nil
}

func (a *SQLiteAdapter) Close() error {
    if a.db != nil {
        return a.db.Close()
    }
    return nil
}

func (a *SQLiteAdapter) GetSlowQueries(ctx context.Context, opts QueryOpts) ([]models.SlowQuery, error) {
    // Query sqlite_stat1 or custom logging table
    return nil, fmt.Errorf("not yet implemented")
}

func (a *SQLiteAdapter) GetExplainPlan(ctx context.Context, query string, opts QueryOpts) (*models.ExplainPlan, error) {
    // Use EXPLAIN QUERY PLAN
    return nil, fmt.Errorf("not yet implemented")
}

func (a *SQLiteAdapter) GetIndexStats(ctx context.Context) ([]models.IndexStat, error) {
    // Query sqlite_master
    return nil, fmt.Errorf("not yet implemented")
}

func (a *SQLiteAdapter) GetTableStats(ctx context.Context) ([]models.TableStat, error) {
    return nil, fmt.Errorf("not yet implemented")
}

func (a *SQLiteAdapter) GetDuplicateIndexes(ctx context.Context) ([]models.DuplicateIndex, error) {
    return nil, fmt.Errorf("not yet implemented")
}

func (a *SQLiteAdapter) GetDatabaseStats(ctx context.Context) (*models.DatabaseStats, error) {
    return nil, fmt.Errorf("not yet implemented")
}
```

### 3. Register in the factory

Edit `internal/adapter/adapter.go`, add the `"sqlite"` case:

```go
func NewAdapter(dbType string) (DBAnalyzer, error) {
    switch dbType {
    case "postgres":
        return NewPostgresAdapter(), nil
    case "mysql":
        return NewMySQLAdapter(), nil
    case "mariadb":
        return NewMariaDBAdapter(), nil
    case "sqlite":
        return NewSQLiteAdapter(), nil
    default:
        return nil, fmt.Errorf("unsupported db type: %s", dbType)
    }
}
```

### 4. Update the frontend connection form

In `apps/web/src/components/connections/connection-form.tsx`, add `"sqlite"` to the database type options:

```typescript
const dbTypeOptions = [
  { value: "postgres", label: "PostgreSQL" },
  { value: "mysql", label: "MySQL" },
  { value: "mariadb", label: "MariaDB" },
  { value: "sqlite", label: "SQLite" },
]
```

## Testing Your Adapter

Write tests in `internal/adapter/mysql_test.go`. Integration tests require a real MySQL instance — use `testcontainers-go` or a Docker Compose service:

```go
func TestMySQLAdapter_Connect(t *testing.T) {
    dsn := os.Getenv("TEST_MYSQL_DSN")
    if dsn == "" {
        t.Skip("TEST_MYSQL_DSN not set")
    }
    a := NewMySQLAdapter()
    err := a.Connect(context.Background(), dsn)
    require.NoError(t, err)
    defer a.Close()
}
```

Run with:

```bash
TEST_SQLITE_DSN="file::memory:?cache=shared" go test ./internal/adapter/ -v -run TestSQLite
```

## Learning from MySQL/MariaDB Implementations

For detailed reference, study the MySQL and MariaDB adapter implementations:

1. **Performance schema conversion**: See how MySQL's picosecond timers are converted to milliseconds in `internal/adapter/mysql_slow_queries.go`
2. **EXPLAIN parsing**: Both MySQL and MariaDB have custom JSON parsing logic in their `*_explain.go` files
3. **Duplicate index detection**: MariaDB's `mariadb_indexes.go` uses `JSON_ARRAYAGG` for grouping — a useful pattern for databases without `array_agg`
4. **Tests**: Both `mysql_explain_test.go` and `mariadb_explain_test.go` demonstrate how to test JSON parsing with real output examples

## PR Checklist

Before opening a pull request for a new adapter:

- [ ] All 8 interface methods implemented (no permanent `fmt.Errorf("not yet implemented")` stubs)
- [ ] Unit tests cover `Connect`, `Close`, and at least `GetSlowQueries`
- [ ] Integration test with a real database instance (skipped when env var absent)
- [ ] Factory case added in `NewAdapter()` in `internal/adapter/adapter.go`
- [ ] Frontend database type option added in `connection-form.tsx`
- [ ] Documentation added or updated under `apps/docs/` with database-specific setup instructions
- [ ] SQL queries verified to work with the target database version
- [ ] DSN format documented with examples
